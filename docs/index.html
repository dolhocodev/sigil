Workflow

From idea to sealed agreement
in under 2 minutes.

1

Initialize

Run sigil init and choose a template. Fill in parties, scope, and expiry interactively.

2

Hash

Sigil computes a SHA-256 content hash — your tamper-proof anchor. The hash is embedded in the document.

3

Sign

Each party runs sigil sign <id>. Your GPG key signs the content hash. Works without GPG too (hash-only mode).

4

Verify

Run sigil verify <id> anytime. Checks hash integrity and all GPG signatures simultaneously.

5

Publish

Run sigil publish <id> to commit to git history and open a PR to the public registry.

6

Immutable

GitHub Actions auto-verifies on every push. The agreement is now permanently sealed in git history.

Protocol

Mathematically guaranteed.

Git's DAG structure is cryptographically equivalent to a permissioned blockchain — at zero operational cost.

H(commit) = SHA256(tree ∥ parent ∥ author ∥ timestamp ∥ message)

Valid Agreement ⟺
  ∀ party ∈ parties:
    verify_gpg(party.signature, SHA256(agreement_text)) = true
  ∧ contentHash = SHA256(current_file_content)
  ∧ commit is reachable from both parties' local clone

Tamper Detection:
  Modify any byte → SHA256 changes → all GPG sigs invalid
  Complexity of forgery: 2^256 operations (infeasible)
